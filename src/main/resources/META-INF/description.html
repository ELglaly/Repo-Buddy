RepoBuddy: Run, Trace, and Inspect Spring Data Repositories in IntelliJ IDEA

RepoBuddy gives Spring Boot teams a faster way to understand and validate Spring Data JPA behavior directly inside IntelliJ IDEA. Execute repository methods from the editor, inspect generated SQL, review returned data, and trace endpoint-to-repository flows without leaving the IDE.

Instead of switching between tests, REST tools, logs, and debugger sessions, RepoBuddy keeps repository execution and analysis in one focused workflow for everyday development and debugging.

Core capabilities

  • Repository runner from the editor - launch repository methods directly from a gutter action.
  • Live execution against the real application - run against your active Spring Boot runtime, not a simulated environment.
  • Structured parameter forms - generate input controls based on repository method parameter types.
  • Random sample data generation - populate parameters quickly with sensible generated values.
  • SQL inspection - view the exact Hibernate SQL statements triggered by execution.
  • Formatted result viewer - inspect returned JSON, execution time, and runtime errors in one place.
  • Repository Usage dashboard - review repository methods, call counts, unused entries, and operation types across the project.
  • Call Chain Tracer - trace controller and service flows down to repository access with transaction-aware context.

Why RepoBuddy

  • Reduce the edit-run-debug cycle when validating persistence logic.
  • See repository behavior in the real runtime with the real SQL output.
  • Understand usage and impact before refactoring repository-heavy code.
  • Keep execution, tracing, and inspection inside IntelliJ IDEA.

Typical workflow

  1. Start your Spring Boot application from IntelliJ IDEA.
  2. Open a Spring Data repository interface.
  3. Trigger RepoBuddy from the repository method gutter action.
  4. Enter values or generate sample input.
  5. Review SQL logs, returned data, execution time, and errors in one popup.

Best suited for

  • Spring Boot developers debugging JPA and repository behavior.
  • Teams working with complex queries, filtering, pagination, and transactional flows.
  • Developers reviewing or refactoring data-access codebases.

Requirements

  • IntelliJ IDEA 2023.2 or later.
  • Java 17 or later.
  • Spring Boot with Spring Data JPA.
  • An application launched from IntelliJ IDEA.
